# KYC Verification

KYC (Know Your Customer) verification is required to enable live payment processing. This guide explains the verification process and requirements.

## Overview

Primeflow operates in two modes:

| Mode | Description | Requirements |
|------|-------------|--------------|
| **Test Mode** | Sandbox environment for development | None - available immediately |
| **Live Mode** | Production payments with real money | KYC verification required |

<Callout type="info">
  All new organizations start in test mode. Complete KYC verification to enable live payments.
</Callout>

---

## Verification Process

<Steps>
  <Step>
    ### Submit Business Information
    
    Provide your business details through the dashboard:
    - **Business Name**: Legal registered name
    - **Registration Number**: Business incorporation/registration number
    - **Business Address**: Registered address
    - **Country**: Country of incorporation
    - **Website** (optional): Company website URL
  </Step>
  
  <Step>
    ### Review Period
    
    Our team reviews submissions within 1-2 business days. You'll receive email notifications for:
    - Submission confirmation
    - Approval notification
    - Rejection with feedback (if applicable)
  </Step>
  
  <Step>
    ### Live Mode Enabled
    
    Once approved, your organization automatically gains access to:
    - Live payment processing
    - Real money transactions
    - Production webhooks
  </Step>
</Steps>

## KYC Status

Your organization's KYC status can be one of:

| Status | Description |
|--------|-------------|
| `pending` | KYC not yet submitted |
| `submitted` | Under review |
| `approved` | Verified - live mode enabled |
| `rejected` | Rejected - see feedback and resubmit |

### Check Status via API

```bash
curl -X GET https://api.primeflow.com/v1/kyc/status \
  -H "X-Org-Id: your_org_id" \
  -H "Cookie: primeflow_session=your_session"
```

**Response:**
```json
{
  "org_id": "org_abc123",
  "kyc_status": "approved",
  "live_enabled": true
}
```

---

## Submit KYC via API

```bash
curl -X POST https://api.primeflow.com/v1/kyc/submit \
  -H "Content-Type: application/json" \
  -H "X-Org-Id: your_org_id" \
  -H "Cookie: primeflow_session=your_session" \
  -d '{
    "business_name": "Acme Payments Ltd",
    "registration_no": "CS123456789",
    "business_address": "123 Business Street, Accra, Ghana",
    "business_country": "GH",
    "website": "https://acmepayments.com"
  }'
```

## Email Notifications

You'll receive email notifications at key stages:

- **Submission Received**: Confirmation that your KYC has been submitted
- **KYC Approved**: Your business is verified and live mode is enabled
- **KYC Rejected**: Feedback on why verification failed and next steps

## Organization Types

### Personal Organizations

- Single-user organizations
- Limited to test mode only
- Cannot invite team members
- Upgrade to business for full features

### Business Organizations

- Multi-user team support
- KYC verification available
- Live mode after approval
- Role-based access control

### Upgrading to Business

Personal organizations can upgrade to business via the dashboard or API:

```bash
curl -X POST https://api.primeflow.com/v1/onboarding/upgrade-to-business \
  -H "X-Org-Id: your_org_id" \
  -H "Cookie: primeflow_session=your_session"
```
